# 媒体查询与响应式策略

## 移动优先

```css
/* 默认：移动端 */
.container {
  padding: 12px;
}
/* 断点增强：平板 */
@media (min-width: 640px) {
  .container {
    padding: 16px;
  }
}
/* 桌面 */
@media (min-width: 1024px) {
  .container {
    padding: 24px;
  }
}
```

## 常见断点建议（可按产品调整）

- 640px（sm）：手机 → 平板
- 1024px（lg）：平板 → 桌面
- 1280px（xl）：大屏

## 交互与命中区域

```css
@media (hover: hover) and (pointer: fine) {
  /* 桌面：允许 hover 动效 */
  .btn:hover {
    transform: translateY(-1px);
  }
}
@media (hover: none) and (pointer: coarse) {
  /* 触屏设备：增大可点击区域 */
  .btn {
    padding: 12px 18px;
  }
}
```

## 练习

- 为产品卡片实现三档布局：1 列/2 列/3 列，并为触屏优化点击区域。\*\*\*

## 知识拓展

- `prefers-reduced-data`（节省流量偏好）

```css
@media (prefers-reduced-data: reduce) {
  img[loading="lazy"] {
    filter: blur(2px);
  } /* 只是示例，实际可降级资源 */
}
```

- 媒体特性组合的策略表

- 颜色方案：`prefers-color-scheme`
- 运动偏好：`prefers-reduced-motion`
- 数据节省：`prefers-reduced-data`
- 指针类型：`pointer`、`any-pointer`
- 悬停能力：`hover`、`any-hover`
- 亮度/对比度：`prefers-contrast`（实验）

- 容器查询 vs 媒体查询：组件内适配优先使用容器查询；布局级别使用媒体查询。
