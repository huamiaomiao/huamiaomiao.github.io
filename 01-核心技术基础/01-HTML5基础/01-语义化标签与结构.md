# HTML5 基础：语义化标签与结构

## 为什么要语义化

- 更易读：结构清晰，降低维护成本
- 更易被机器理解：SEO 友好、搜索引擎和读屏器受益
- 更易无障碍：内置地标区域（Landmarks），减少过度使用 `role`

---

## 常见语义化标签速查

- 布局与地标
  - `header`：页头或区块头部（可包含标题、logo、导航）
  - `nav`：导航区域（主导航、面包屑、目录等）
  - `main`：文档主要内容（每页仅一个）
  - `article`：可独立成篇的内容单元（文章、帖子、评论）
  - `section`：主题性分组（需有标题）
  - `aside`：侧栏、附加信息、广告、相关链接
  - `footer`：页尾或区块尾部（版权、补充信息）
- 文本与内容
  - `h1` ~ `h6`：标题，按层级使用
  - `p`：段落；`blockquote`/`q`：引用（长/短）
  - `ul`/`ol`/`li`：无序/有序列表；`dl`/`dt`/`dd`：定义列表
  - `figure`/`figcaption`：带标题的独立媒体（图表、代码等）
  - `time`：时间（结合 `datetime` 机器可读）
  - `address`：联系信息（作者或组织）
  - `em`/`strong`：语义强调；`mark`：高亮；`small`：次要信息
  - `code`/`pre`/`kbd`/`samp`：代码、预格式文本、键盘输入、示例输出
  - `abbr`（`title`）/`cite`/`dfn`：缩写、引用来源、术语定义
  - `data`（`value`）：机器可读值绑定到可见文本

## 页面语义骨架示例

```html
<!DOCTYPE html>
<html lang="zh-CN">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width,initial-scale=1" />
    <title>示例页面</title>
  </head>
  <body>
    <header>
      <h1>站点标题</h1>
      <nav aria-label="主导航">
        <ul>
          <li><a href="/">首页</a></li>
          <li><a href="/blog">博客</a></li>
          <li><a href="/about">关于</a></li>
        </ul>
      </nav>
    </header>

    <main id="main-content">
      <article>
        <header>
          <h2>文章标题</h2>
          <p><time datetime="2025-11-17">2025年11月17日</time> · 作者张三</p>
        </header>

        <section>
          <h3>小节标题</h3>
          <p>段落内容……</p>
          <figure>
            <img src="./image.png" alt="学习路径总览" />
            <figcaption>图1：学习路径示意</figcaption>
          </figure>
        </section>

        <section>
          <h3>另一小节</h3>
          <p>更多内容……</p>
          <pre><code class="language-html">&lt;div&gt;代码片段&lt;/div&gt;</code></pre>
        </section>

        <footer>
          <p>标签：<a href="/tags/html5">HTML5</a></p>
        </footer>
      </article>

      <aside aria-label="侧栏">
        <section>
          <h2>相关推荐</h2>
          <ul>
            <li><a href="#">语义化与可访问性</a></li>
            <li><a href="#">SEO 基础</a></li>
          </ul>
        </section>
      </aside>
    </main>

    <footer>
      <address>联系：contact@example.com</address>
      <small>© 2025 示例公司</small>
    </footer>
  </body>
</html>
```

## `article` / `section` / `div` 选择

- `article`：内容自包含、可单独分发（博客文章、卡片、评论）
- `section`：主题性分组，建议有可见标题（`h2+`）
- `div`：无语义容器，仅用于样式或脚本钩子（无合适语义时再用）

---

## 标题层级与文档大纲

- 页面应有且仅有一个主标题语义（通常 `h1` 在 `header` 或 `main` 内）
- 小节标题层级递进：`h2` > `h3` > `h4`…
- 避免跳级（从 `h2` 跳到 `h4`）

示例：

```html
<main>
  <h1>页面主标题</h1>
  <section>
    <h2>一级主题</h2>
    <h3>二级主题</h3>
  </section>
</main>
```

---

## 可访问性与 SEO 要点

- 地标区域：`header/nav/main/aside/footer` 提供天然 Landmarks
- 图像 `alt` 文本准确表达含义；装饰性图像用空 `alt=""`
- 表单语义：`label for`、`fieldset/legend`、合适的 `type` 与 `required`
- 链接文本可读且具描述性（避免“点击这里”）
- 使用 `time[datetime]`、`address`、`abbr[title]`、`figure/figcaption` 提升机器可读性
- 有意义再补充 `role`，不要与原生语义冲突

---

## 表单语义示例

```html
<form>
  <fieldset>
    <legend>联系信息</legend>
    <label for="email">邮箱</label>
    <input id="email" type="email" required />

    <label for="tel">电话</label>
    <input id="tel" type="tel" inputmode="numeric" />
  </fieldset>

  <button type="submit">提交</button>
</form>
```

---

## 常见反模式

- 用大量 `div` 模拟结构，忽略现成语义标签
- 用标题标签做纯样式（如只为变大/变粗）
- 跳级或重复多个 `h1`
- 链接当按钮用（应使用 `button` 触发行为），或按钮当链接用
- 图像缺少 `alt`，表单缺少 `label`

---

## 小清单（Checklist）

- [ ] 首先选对语义标签，再写样式
- [ ] 标题层级严格递进，页面单一主标题
- [ ] 主要区域用地标元素包裹：`header/nav/main/aside/footer`
- [ ] 图片 `alt`、表单 `label`、时间 `time[datetime]`
- [ ] 只在必要时添加 `role` 或 ARIA 属性

## 知识拓展

- 面包屑导航语义

```html
<nav aria-label="面包屑">
  <ol>
    <li><a href="/">首页</a></li>
    <li><a href="/docs">文档</a></li>
    <li aria-current="page">HTML 语义化</li>
  </ol>
  <style>
    nav[aria-label="面包屑"] ol {
      display: flex;
      gap: 0.5rem;
      list-style: none;
      padding: 0;
      margin: 0;
    }
    nav[aria-label="面包屑"] li + li::before {
      content: "/";
      opacity: 0.6;
      margin-inline: 0.25rem;
    }
  </style>
</nav>
```

- 列表/卡片的正确语义

```html
<section aria-labelledby="products-title">
  <h2 id="products-title">热门产品</h2>
  <ul class="product-list">
    <li>
      <article>
        <h3><a href="/p/1">智能音箱 S1</a></h3>
        <p>支持语音助手与家庭互联</p>
      </article>
    </li>
    <li>…</li>
  </ul>
</section>
```

- 表格语义增强（表头/描述）

```html
<table>
  <caption>
    季度营收
  </caption>
  <thead>
    <tr>
      <th scope="col">季度</th>
      <th scope="col">营收（万元）</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Q1</th>
      <td>1200</td>
    </tr>
    <tr>
      <th scope="row">Q2</th>
      <td>1380</td>
    </tr>
  </tbody>
</table>
```
