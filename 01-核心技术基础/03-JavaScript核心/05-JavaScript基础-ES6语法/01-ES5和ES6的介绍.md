# ES5 和 ES6 的介绍

> 目标：了解 ES 的发展、差异与迁移方向，为后续语法学习建立全局观。

## ECMAScript 简史

- ES3（1999）：早期基础
- ES5（2009）：添加严格模式、JSON、数组方法、Object API 等
- ES6/ES2015（2015）：一次大升级，后续采用“每年一个版本”的节奏（ES2016+）

## 为什么需要 ES6

- 模块化、类、块级作用域、箭头函数、解构、Promise、迭代器、集合类型等
- 提升可读性与工程化能力，减少“工具/库补洞”

## 兼容性

- 现代浏览器基本支持；旧环境可用 Babel/TypeScript 转译
- 新项目优先使用 ES6+；维护老项目时理解 ES5 有帮助

## 学习路线建议

1. 作用域与变量 → 解构 → 函数与 this → 集合与迭代
2. 模块化与工程工具（Vite/Babel）
3. 异步（Promise/async）与模块按需加载
