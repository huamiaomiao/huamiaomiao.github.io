# ES5 中的严格模式（"use strict"）

> 目标：理解严格模式的作用、常见报错与迁移建议。

## 开启方式

```js
"use strict"; // 文件/函数顶部
function demo() {
  "use strict";
}
```

## 主要变化

- 禁止意外的全局变量：未声明直接赋值会报错
- `this` 在函数中默认是 `undefined`（非构造/非绑定）
- 禁止重复参数名/重复属性名
- `with` 语句被禁用
- `eval` 更安全：在其内部创建的变量不会泄露到外层作用域

## 示例

```js
"use strict";
function f() {
  // x = 1; // ReferenceError（必须显式声明）
  return this; // undefined
}
```

## 迁移建议

- 新项目默认使用严格模式（模块/ES6 默认近似严格）
- 打开 linter（ESLint）避免隐式全局、重复定义等问题
