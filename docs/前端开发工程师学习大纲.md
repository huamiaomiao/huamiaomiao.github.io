## 前端开发工程师学习大纲·0-1

### 一、岗位介绍

- 定义：面向浏览器/小程序等运行环境，负责界面呈现、交互逻辑与用户体验的工程师
- 核心职责
  - 将视觉与交互方案落地为高质量页面与组件
  - 与后端联调，完成数据对接与状态管理
  - 保障性能、可用性、可访问性与安全
  - 搭好开发基础设施（项目模板、代码规范、自动化构建发布、线上监控）
- 工作边界（常见协作分工）
  - 与产品/设计：需求澄清、交互细节、验收标准
  - 与后端：接口定义、错误约定、性能与稳定性
  - 与测试/运维：测试策略、发布流程、监控与回滚
- 价值与影响
  - 直接影响用户体验与业务转化
  - 推动团队研发效率与质量提升
  - 形成可复用的工程资产（组件库、规范、工具）
- 岗位画像（简述）
- 初级：能按规范独立完成模块与基础联调
- 中级：能负责中小型功能/页面、优化性能与工程质量
- 高级及以上：能主导架构与工程体系，驱动跨团队协作与落地

### 二、工作内容

- 端到端工作流（标准步骤与产物）
  - 需求澄清
  - 方案评审
  - 环境搭建
  - 开发实现
  - 联调与自测
  - 代码评审
  - 测试与修复
  - 构建与发布
  - 监控与迭代
- 日常协作与工具
  - 版本控制：Git 分支模型、PR、Code Review 规范
  - 任务管理：飞书/禅道，需求拆分与估时
  - 文档沉淀：README、Design Doc、接口与约定、变更日志
  - 沟通机制：站会、评审会、复盘会，问题升级路径
- 交付物与质量门槛
  - 代码规范：ESLint/Prettier/Commitlint、提交信息规范
  - 测试覆盖：单测/组件测试/E2E 的合理覆盖率
  - 性能预算：LCP/CLS/TTI/SI 等指标达标
  - 可访问性：键盘可达、语义化、颜色对比、ARIA 标签
  - 安全基线：XSS/CSRF 防护与依赖漏洞治理
- 典型工作场景
- 新功能开发、遗留系统重构、性能专项治理、设计还原
- 线上故障快速定位与回滚、灰度发布与监控告警处理

### 三、具备技能

- Web 基础
  - HTML：语义化、可访问性（A11y/ARIA）、SEO 友好
  - CSS：Flexbox/Grid、响应式、选择器优先级、变量/函数、动画与过渡、BEM 命名
  - JavaScript/TypeScript：类型系统、异步（Promise/async/await）、模块化、错误处理、函数式与面向对象
- 框架与生态
  - Vue 或 React（建议二选一深入，理解组件化思想与渲染机制）
  - 状态管理：Pinia/Redux（模块化、异步流、数据归一化）
  - 路由：Vue Router/React Router（动态路由、守卫、懒加载）
  - 组件设计：可复用性、可测试性、无障碍、API 设计与文档
  - SSR/SSG：Nuxt/Next 的基本概念与适用场景
- 工程化
  - 构建：Vite/Webpack 的打包与性能优化，代码分割与懒加载
  - 包管理：npm/pnpm 的工作空间、锁文件与依赖安全
  - 规范：ESLint/Prettier/Commitlint/Husky/Lint-Staged
  - DevOps：CI/CD（GitHub Actions/GitLab CI）、版本与变更日志、发布与回滚
- 质量与测试
  - 单元测试：Vitest/Jest（覆盖率、快照、Mock）
  - 组件测试：Testing Library（以用户视角编写用例）
  - 端到端测试：Cypress/Playwright（关键路径保障）
  - 可观察性：日志、前端监控（性能/错误）、埋点与分析
- 网络与性能
  - 协议：HTTP/HTTPS、CORS、缓存策略、CDN
  - 性能指标：LCP/CLS/FID/TTI/SI，RUM 收集与诊断
  - 优化手段：图片与字体优化、Preload/Prefetch、Tree Shaking、代码分割
- 安全与稳定
  - 常见风险：XSS/CSRF/Clickjacking/依赖漏洞
  - 防护策略：转义/内容安全策略（CSP）/SameSite Cookie/权限与鉴权
- 计算机基础与软技能
  - 浏览器原理：渲染管线、事件循环、存储（Cookie/Storage/IndexedDB）
  - 数据结构与算法：常见结构与时间/空间复杂度
  - 软技能：需求澄清、技术写作、跨团队沟通、时间管理与复盘
- 学习路径（0→1）
  - 入门：Web 三件套 + 一个框架基础 + Git 基本功
  - 进阶：状态管理/路由/组件设计 + 异步与网络
  - 工程化：构建工具、规范、测试、CI/CD、监控
  - 专项：性能优化、安全治理、可访问性、SSR/SSG

### 五、项目实战：从 0-1 搭建个人博客（初级前端版）

- 1. 立项与范围
  - 明确目标：写技术/生活文章，先做 MVP（文章列表、文章详情、关于我、标签/分类）
  - 成功标准：可访问、可发布新文、移动端可用、加载速度可接受
  - 里程碑：MVP（本周）、基础 SEO/统计（下周）、评论与搜索（后续）
- 2. 技术选型与账号
  - 站点形态：静态站（Hexo/Hugo/VitePress，入门快）或框架站（Vite + Vue/React，可扩展）
  - 托管平台：Vercel/Netlify/GitHub Pages（三选一，若由后端/运维统一部署可忽略）
  - 账号准备：GitHub 账号、（可选）域名与 DNS 服务
  - 本项目选择：Vite + Vue 3 + Vue Router（后续步骤以此为准）
- 3. 仓库与规范
  - 创建仓库：新建 Git 仓库并初始化分支（main、dev）
  - 基础规范：ESLint + Prettier（保证风格统一）、EditorConfig（编辑器一致）
  - 提交规范（可选）：Commitlint + Husky（保证提交信息可读）
- 4. 初始化与目录结构
  - 使用 Vite 脚手架初始化 Vue 3 + TypeScript 项目并启动开发服务

```bash
npm create vite@latest my-blog -- --template vue-ts
cd my-blog
npm install
npm run dev
```

- 目录示例：

```text
posts/                # Markdown 文章
src/                  # 站点源码（组件/页面/样式）
  components/
  pages/
  assets/
public/               # 静态资源（图标、站点图等）
```

- 5.  基础架构与页面（Vue 3 + Vite）
  - 安装路由（Vue Router 4）

```bash
npm i vue-router@4
```

- 创建 `src/router/index.ts`，配置路由：`/`（列表）、`/post/:slug`（详情）、`/tags`、`/about`
- 页面与组件：`src/pages/{Home.vue,Post.vue,Tags.vue,About.vue}`；通用组件：`src/components/{NavBar.vue,PostCard.vue,TagList.vue,Footer.vue}`
- 在 `App.vue` 组织布局：Header（导航）/Main（内容区）/Footer（页脚）
- 在 `main.ts` 注册路由：`app.use(router).mount('#app')`
- 路由模式：默认使用 `createWebHistory`；如纯静态托管未配置回退，可改用 `createWebHashHistory`
- 6.  内容模型与数据（前端本地）
  - 文章格式：Markdown + Front Matter（title/date/tags/summary/cover）
  - 文件组织：`posts/2025-01-my-first-post.md`（建议英文 slug）；封面图放在 `public/cover/`
  - 安装解析库（浏览器可用）

```bash
npm i marked gray-matter
```

- 示例文章（放入 `posts/hello-world.md`）

```markdown
---
title: Hello World
date: 2025-01-01
tags: [intro]
summary: 我的第一篇博客
cover: /cover/hello.jpg
---

这里是 Markdown 正文内容...
```

- 在页面中加载与解析（示例）

```ts
// 示例：src/pages/Post.vue 中按 slug 加载
import { marked } from "marked";
import matter from "gray-matter";
// vite 支持 ?raw 直接以字符串方式导入
// 实际项目中可通过 import.meta.glob 批量导入
const files = import.meta.glob("../../posts/*.md", { as: "raw" });

async function loadPost(slug: string) {
  const key = Object.keys(files).find((k) => k.includes(`${slug}.md`));
  if (!key) return null;
  const raw = await files[key](); // 获取 markdown 字符串
  const { data, content } = matter(raw);
  const html = marked.parse(content);
  return { meta: data, html };
}
```

- 7.  实现 MVP 页面
  - 列表页（`Home.vue`）：用 `import.meta.glob` 批量读取 `posts/*.md`，解析出 meta 列表，按时间倒序，分页（可后置）
  - 详情页（`Post.vue`）：根据路由 `:slug` 加载并解析 Markdown，支持代码高亮（可后置）
  - 标签页（`Tags.vue`）：根据 meta.tags 聚合过滤
  - 关于页（`About.vue`）：基本介绍、社交链接、邮箱
- 8.  基础功能与体验
  - 响应式：移动端适配（断点：375/768/1200）
  - 图片：封面图按比例裁剪、懒加载、构建阶段压缩（vite-imagetools 可选）
  - SEO 基础：每页设置 title/description；后续再补 sitemap/robots（可选）
  - 代码高亮（可选）：`npm i prismjs` 并在详情页引入样式
  - 图片工具（可选）：`npm i -D vite-imagetools` 并在 `vite.config.ts` 中开启插件

```ts
// vite.config.ts
import { defineConfig } from "vite";
import vue from "@vitejs/plugin-vue";
import { imagetools } from "vite-imagetools";

export default defineConfig({
  plugins: [vue(), imagetools()],
});
```

- 9.  测试与自检（轻量）
  - 自检清单：路由可达；内外链不 404；图片加载正常；移动端无明显错位
  - 冒烟测试：打开主页 → 进入详情 → 返回 → 切换标签 → 关于页
  - 无障碍基础：键盘可聚焦、图片有 alt、语义化标签
- 10. 构建与本地预览（前端完结点）
  - 构建产物

```bash
npm run build
```

- 本地预览构建结果（可验证路由与资源）

```bash
npm run preview
```

- 构建输出在 `dist/`，交付给后端或运维部署即可
- 11. 监控与统计（可后置）
  - 访问统计：后续接入 Umami/GA
  - 错误监控：必要时接入 Sentry，关注 404 与脚本报错
  - 性能关注：首屏加载时间与图片体积
- 12. 运营与迭代
  - 内容工作流：本地写作 → 预览 → 提交 PR → 合并后打包产出 `dist/`，由后端/运维部署
  - 备份与协作：文章与图片入库 Git，重要资源定期备份
  - 迭代计划：收集反馈，规划搜索/评论/PWA/主题优化等 Backlog
